## Analysis

### Context Referencing (77% avg)
The model maintained good context awareness across turns. Notably:
- Turns 1-2, 8-9: 100% — perfect recall of immediately prior context
- Turns 3-4: 33-67% — "backoff" keyword not referenced when renamed to
  max_retries/timeout (naming divergence, not context loss)
- Turn 10: 50% — "circuit" and "retry" not mentioned in test update response

### Why Results Are Identical
1. Gemini 3.1 Flash-Lite has 1M token context — no pressure
2. Done_messages grew linearly (4 → 36), never triggering summarizer.too_big()
3. Same messages sent = same (deterministic temp=0) responses

### When BM25 Would Differentiate
The BM25 patch would show value when:
- Context window < 32K tokens (e.g., older models, cost-constrained)
- Sessions > 30 turns (done_messages grows large enough to trigger compaction)
- Multiple large files in chat (repo indexing fills context)

With a smaller context budget, stock Aider would summarize old messages using
an LLM (costing tokens + losing detail), while BM25 would deterministically
select the most relevant messages (zero cost, preserves exact text).
